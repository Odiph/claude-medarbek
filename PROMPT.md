# פרומפט ל-Claude Code — בניית הפרויקט "claude-medarbek"

העתק את כל הקובץ הזה והדבק אותו ל-Claude Code הלוקאלי שלך (או שמור אותו
כ-`PROMPT.md` בתיקייה ריקה ותגיד ל-Claude "בצע את ההוראות ב-PROMPT.md").

═══════════════════════════════════════════════════════════════════════
## מה לבנות

כלי קטן וקוד-פתוח בשם **claude-medarbek** שמחליף את פעלי ה"חשיבה"
שמופיעים ליד הספינר של Claude Code (במקור: Thinking…, Simmering…) בפעלים
עבריים שנונים: מדרבק…, מכווצץ…, מהלוצינט…

## איך זה עובד (מאומת מול קוד ה-CLI, גרסה 2.1.x)

Claude Code תומך רשמית בהגדרה `spinnerVerbs` ב-settings.json. הסכימה המדויקת:

```json
{ "spinnerVerbs": { "mode": "replace" | "append", "verbs": ["..."] } }
```

- `mode: "replace"` — משתמש רק בפעלים שלך.
- `mode: "append"` — הפעלים שלך + ברירות המחדל של Claude.

ההגדרה נקראת מהיררכיית settings.json:
- משתמש (גלובלי): `~/.claude/settings.json`
- פרויקט: `.claude/settings.json`
- מקומי: `.claude/settings.local.json`

הערה: מנגנון הפלאגינים (skills/agents/hooks/MCP) לא חושף שדה לדריסת פעלי
הספינר, ולכן הנתיב הנתמך הוא settings.json — וזה מה שהכלי הזה עושה.

## מפרט הקוד

### lib/apply.mjs
מייצא `applyVerbs({ settingsPath, mode, verbs })` ו-`MODES = ["append","replace"]`.
- קורא settings.json קיים (או {} אם חסר/ריק); זורק שגיאה עם `code:"EBADJSON"`
  אם ה-JSON פגום — כדי לא לדרוס קובץ קיים.
- מציב `settings.spinnerVerbs = { mode, verbs }` בלי לגעת בשאר המפתחות.
- יוצר תיקיות אם צריך וכותב עם 2-space indent + newline סופי.

### bin/cli.mjs
`#!/usr/bin/env node`. טוען verbs.json (או verbs.fem.json עם `--feminine`).
דגלים:
- `--global` (ברירת מחדל) → `~/.claude/settings.json`
- `--project` → `./.claude/settings.json`
- `--append` / `--replace` (ברירת מחדל replace)
- `--feminine` → טוען verbs.fem.json
- `--print` → מדפיס את בלוק ה-JSON ויוצא (לא כותב)
- `-h/--help`
בסיום כותב "✓ Wrote N verbs to <path> (mode)" ומזכיר לפתוח סשן חדש.

### test/smoke.mjs (ללא framework)
בודק: (1) כתיבה לקובץ חדש, (2) שמירת מפתחות קיימים במיזוג, (3) JSON פגום
לא נדרס וזורק EBADJSON, (4) mode לא חוקי נזרק, (5) verbs.fem.json באותו
אורך כמו verbs.json וכל צורה מסתיימת ב-ת או ה.

## ההוראה שלך ל-Claude Code

1. צור את כל הקבצים לפי המבנה והמפרט למעלה, עם רשימות הפעלים המדויקות.
2. הרץ `npm test` וודא שהכל עובר.
3. הרץ `node bin/cli.mjs --print` והראה לי את הפלט.
4. אם אני מאשר — הרץ `node bin/cli.mjs` כדי להחיל את הפעלים גלובלית,
   ואמור לי לפתוח סשן חדש של Claude Code.

## דחיפה ל-GitHub

5. אתחל git וקמט:
   ```bash
   git init -b main
   git add -A
   git commit -m "Initial commit: claude-medarbek"
   ```
6. צור repo ציבורי חדש ודחוף. עם GitHub CLI (הכי מהיר):
   ```bash
   gh repo create claude-medarbek --public --source=. --remote=origin --push
   ```
   או ידנית — צור repo ריק ב-github.com (בלי README), ואז:
   ```bash
   git remote add origin https://github.com/<USERNAME>/claude-medarbek.git
   git push -u origin main
   ```
7. אם שם המשתמש/repo שונה מ-`Odiph/claude-medarbek`, עדכן את ה-URLs
   ב-`package.json` וב-`README.md` (homepage/repository/bugs והתגים).

## פרסום ל-npm (אופציונלי)

8. כדי ש-`npx claude-medarbek` יעבוד לכולם:
   ```bash
   npm login
   npm publish
   ```
   אחרי הפרסום אפשר לתייג גרסה: `git tag v1.0.0 && git push --tags`.
