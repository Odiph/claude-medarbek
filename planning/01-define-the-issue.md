# Goal 1 — Define the Issue

The issue is the **problem space** we commit to. Get this wrong and every use case underneath it is wasted. Get it right and it's stable for years while we swap use cases.

A good issue is: **specific about who** · **about a real, recurring pain** · **not a solution in disguise** · **big enough to matter, narrow enough to own.**

---

## Step 1 — Name the customer, not "everyone"
An issue belongs to a *specific person in a specific context*. "Communities" is not a customer. "The volunteer HOA board member who runs events for a 200-home community" is.

> Our customer is: **[role]** in the context of **[situation]**.

## Step 2 — Find the recurring friction (their words)
- What do they repeatedly struggle with?
- What do they hack around with spreadsheets, group chats, paper?
- Where does the same pain show up every month/quarter?

> The recurring friction is: ____________________

## Step 3 — Say why the status quo is costly
If the pain has no cost, there's no issue. Cost = time, money, missed outcomes, or risk.

> Today this costs them: ____________________

## Step 4 — Write the issue statement
Fill one. Only one.

> **[Customer]** repeatedly struggles to **[recurring job]**, because **[friction]**. Today they cope with **[workaround]**, which costs them **[time/money/risk]**.

**Solution-in-disguise check:** if the sentence contains "an app that…" or "a widget that…", delete it and re-describe the *pain*. The issue must be true even if we never build anything.

---

## Step 5 — Pressure-test the issue (all four must pass)

| Test | Question | Pass? |
|------|----------|:---:|
| **Real** | Can we point to someone who has this pain this month? | |
| **Ours** | Is this a space we can credibly win and want to live in for years? | |
| **Recurring** | Does it happen often enough to build a business on? | |
| **Big enough** | Are there many customers with this same issue? | |

If any fails, the issue is wrong — fix it before moving to use cases. **A weak issue can't be rescued by a clever use case.**
